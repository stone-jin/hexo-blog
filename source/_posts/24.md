## 一、介绍
Spring Security是springboot里面针对权限校对的一遍便捷的方式，主要是我们使用springboot编写的web项目，要能权限访问的时候常用。比如写了一个管理系统，那肯定是需要让这个管理系统能进行权限校验才能进行访问，否则这个系统里面的数据都被暴露给别人了，那这个系统的安全性就没有了。

## 二、解决办法
方案一：通过aop我们自己手写一个权限校验

方案二：使用Spring security是一个比较方便的方法

方案三：使用shrio来处理

## 三、本文重点Spring security
3.1 引入pom包
```text
<dependency>
   <groupId>org.springframework.boot</groupId>
   <artifactId>spring-boot-starter-security</artifactId>
</dependency>
```

然后引入pom包，我们需要在application.yml文件中进行配置对应的账号密码

```text
security:
  user:
    name: admin
    password: 123456
```

然后如果外面是spring boot2.0里面的Spring security配置项：

```text
spring:
  security:
    user:
      name: stone-jin
      password: stone-jin
```

我们在我们的springboot的代码中编写我们的controller来进行测试，比如我们此处编写一个hello.do接口进行测试：

```text
@RestController
@RequestMapping("/xhr/test")
public class testController {

    @GetMapping("/hello.do")
    public ResultVO hello(){
        ResultVO resultVO = new ResultVO();
        resultVO.setCode(200);
        resultVO.setMessage("hello world");
        return resultVO;
    }
}
```

然后编写完毕后，我们运行我们的代码查看我们的效果：

![](https://520stone-blog.oss-cn-beijing.aliyuncs.com/upload/2018-04-27T09:03:17Z/310481cf-8a17-4e12-8ff9-9a9045b8c76f.png)

然后最终我们的目的就达到了，就是比如自己做的管理后台能进行权限的管理了，不是每个用户都能访问了。
